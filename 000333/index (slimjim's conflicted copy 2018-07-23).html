Interpreting uninterpretable features
Peter Svenonius
September 2006
 

I address the question of whether there are uninterpretable features. First, I define the notion, in a way compatible with some recent work. Interface features are those which are in principle interpretable at the interface between two modules, and interpretable features are those which are in fact mapped from a representation in one module to a corresponding representation in another module. Uninterpretable features are interface features which are present only in the representation of one module. I suggest that given this definition, it is an empirical fact that there are uninterpretable features. This supports the idea that syntax is driven, at least in part, by the need to check uninterpretable features.
